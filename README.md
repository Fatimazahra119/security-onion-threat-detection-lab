Security Onion Threat Detection 

 Overview

This project demonstrates the deployment and configuration of Security Onion as a Security Information and Event Management (SIEM) platform for monitoring network activity and detecting security threats.

 Objectives

 Install and configure Security Onion
 Monitor network traffic and system logs
 Detect suspicious activities
 Analyze security events and alerts
 Gain practical experience with SIEM tools

 Environment

 Security Onion
 Linux
 Virtual Machines
 Kibana
 Elasticsearch
 Suricata
 Zeek

 Security Tests Performed

 Test 1: Nmap Scan Detection

Objective: Detect network reconnaissance activity.

Actions:

 Performed an Nmap scan against a target machine.
 Monitored alerts generated by Security Onion.
 Analyzed logs and events through the Security Onion dashboard.

Results:

 Reconnaissance activity was detected.
 Relevant alerts were generated and investigated.

 Test 2: SSH Brute Force Detection

Objective: Detect repeated SSH authentication attempts.

Actions:

 Simulated multiple SSH login attempts in a controlled lab environment.
 Monitored generated alerts and logs.

Results:

 Security Onion detected suspicious authentication activity.
 Security events were successfully analyzed.
